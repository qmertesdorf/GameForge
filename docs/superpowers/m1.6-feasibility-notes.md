## Findings (in the order they surfaced)
1. **Provisional template was structurally wrong for Stable Audio (fixed).** The committed `stable-audio.json` wired `CLIPTextEncode.clip` from the checkpoint's CLIP output and used `scheduler:"normal"`. ComfyUI's own bundled example (`comfyui_workflow_templates/.../audio_stable_audio_example.json`) shows the real graph: the T5 text encoder loads via a **separate `CLIPLoader`** (`clip_name:"t5-base.safetensors"`, `type:"stable_audio"`) feeding **both** positive and negative `CLIPTextEncode`; KSampler uses `scheduler:"exponential"`. No `ConditioningStableAudio` node is needed (seconds come from `EmptyLatentAudio`). **Template corrected to match** (committed). The checkpoint's MODEL (slot 0) and VAE (slot 2) are used; its CLIP (slot 1) is unused.
2. **`torchcodec` blocker → local soundfile patch (fixed).** This ComfyUI build's only save node is `SaveAudio`, which calls `torchaudio.save(..., format="FLAC")`. Under **torch/torchaudio 2.11** (the cu128 build required for Blackwell), `torchaudio.save` now routes through **TorchCodec**, which is not installed (and needs system FFmpeg) → `ImportError: TorchCodec is required`. The graph otherwise ran perfectly (model+T5 load, sample, `VAEDecodeAudio` produced AUDIO) — only the save failed. **Fix:** patched `comfy_extras/nodes_audio.py` `save_audio` to write **WAV via `soundfile`** (bundled libsndfile — no torchcodec, no system FFmpeg). This also resolves the format question: output is real **WAV** (`.wav`), which matches the recipe `format` enum with **no schema change**. (`SaveAudioMP3`/`SaveAudioOpus` exist only in newer ComfyUI than the pinned v0.3.16, so they were not an option without un-pinning the LayerDiffuse-critical version.)
3. **`EmptyLatentAudio` minimum duration is 1.0 s.** SFX recipes must use `duration_s >= 1.0` (the skill's old "0.3–2 s" lower bound is invalid). Probe used 2.0 s.
4. **Format decision = WAV for this milestone.** Pros: Godot-native, lossless, matches the schema enum, zero extra deps. Con: music is uncompressed — the 30 s pad came out **5.29 MB**. Acceptable for a proof; for production, OGG/Vorbis (libsndfile supports it — change the patch's `sf.write` subtype/extension, or transcode in `genAudio`) would cut music ~10×. SFX size is a non-issue (0.36 MB).
