# M1.6 — Audio Asset Skill (local generative SFX + music)

**Date:** 2026-05-31
**Milestone:** M1.6 (audio), following M1 (SVG re-skin) and M1.5 (raster re-skin).
**Status of this doc:** design / spec.

## 1. Goal

Add a new `audio` Claude Agent Skill that takes a `playable`-or-better GameForge game and gives it a coherent, prompt-derived audio identity: discrete sound effects tied to core-loop events plus a looping background music track. Audio is generated locally via **Stable Audio Open 1.0 running in ComfyUI**, wired into the Godot scene, recorded in the manifest as an `audio_pass`, and the game advances to status `scored`.

The real deliverable, as in every prior milestone, is **a better skill** — audio quality is the signal, not the end. Per-run findings fold back into the skill prose.

## 2. Decisions (locked during brainstorming)

| Fork | Decision | Rationale |
|------|----------|-----------|
| Scope | **SFX + music** | Full audio pass; both event SFX and a background loop. |
| Production method | **Local generative model** | Mirrors the M1.5 raster stack; owner chose richness over the determinism/GPU tradeoffs. |
| Serving infra | **Reuse ComfyUI** (extend `tools/comfy.mjs`) | ComfyUI has native audio support and runs Stable Audio Open via graph templates over the same HTTP API we already drive and test. One stack, one seam, one feasibility gate. |
| Skill home | **New `audio` skill** | Audio reasoning (event→SFX mapping, mood→music) is its own domain; keeps `asset` focused on visuals. Shares `comfy.mjs` as infra only. |

## 3. Pipeline placement & status model

- The `audio` skill operates on a game that is at least **`playable`** (the core loop works). It does **not** require `styled` — audio and visual passes are orthogonal and order-independent.
- Terminal audio status is **`scored`**.
- Because `status` is a single string, the manifest's `audio_pass` and `asset_pass` blocks are the **source of truth** for what has actually been done. `scored` implies "playable or better, with audio wired"; whether the game is *also* visually styled is read from `asset_pass`, not inferred from the status string. (Same "mixed-method honesty" principle M1.5 adopted.)

## 4. Determinism contract

Mirrors the M1.5 raster contract, because generative audio has the same property the stack research flagged for images: GPU matmul + seed handling are **not bit-exact**.

- The **rendered audio file (`.wav`/`.ogg`) is canonical and committed to git.**
- The **recipe is provenance, not a bit-exact regeneration spec.** Re-running a recipe yields *a* valid clip, not the identical bytes.
- `.gitattributes` marks audio binaries (`*.wav binary`, `*.ogg binary`).

## 5. Scope contract per game

- **SFX:** one-shot clips bound to core-loop events. The skill derives the event list from `concept.core_loop` + the entity set (e.g. collect, hurt, lose, primary-action). Short (~0.3–2s), `.wav`.
- **Music:** one looping background track matched to game mood, derived from `concept.art_direction` + theme. ~20–40s rendered, `.ogg`, looped via Godot import flags.

## 6. Schema additions

Additive and non-breaking, same strategy as `asset_pass`. No change to top-level `required`. `tools/manifest.mjs` stays untouched (free strings + array-replace already handle this shape).

```
audio_pass:
  audio_system:
    model:            string        # e.g. "stable-audio-open-1.0"
    mood_prompt:      string        # shared mood scaffold across clips (anti-incoherence)
    style_descriptors: [string]     # e.g. ["chiptune","8-bit","upbeat"]
  recipes: [                        # one per clip
    {
      name:            string,
      kind:            "sfx" | "music",
      prompt:          string,
      negative:        string,
      seed:            integer,
      duration_s:      number,
      sampler:         string,
      steps:           integer,
      cfg:             number,
      format:          "wav" | "ogg",
      loop:            boolean,
      import_settings: { loop: boolean, loop_offset: number }
    }
  ]
  events: [                         # the wiring record: what plays when
    { event: string, clip: string, node: string, signal: string }
  ]
```

`origin`/`method` style strings (e.g. `method:"audio"`) remain free strings — no schema change needed for them.

## 7. Tool extension — `tools/comfy.mjs`

Reuse the tested seam in place; do not fork a new tool.

- **Templates:** new audio graph templates under `tools/comfy-templates/` — `stable-audio.json` (Stable Audio Open text→audio graph with `%token%` slots for prompt/negative/seed/duration/steps/cfg). `templateName()` learns to select an audio template when a recipe carries `kind`.
- **`genAudio` path:** alongside the existing image `gen`. Same pure `injectRecipe` (deep-clone + `%token%` substitution), same poll-`/history` → fetch-`/view` loop, but reads the `SaveAudio` output bytes and writes a committed `.wav`/`.ogg`. Same loud-failure contract: throws with host/graph context, writes **no file** on error (guards on `/prompt`, `/history`, `/view`).
- **CLI:** `node tools/comfy.mjs gen-audio <id> <clip-name> '<recipe-json>'`.
- Env vars unchanged (`COMFY_HOST`, `GAMEFORGE_COMFY_TEMPLATES`, `GAMEFORGE_GAMES_DIR`). `maxPolls` ceiling reused (audio gen is far faster than 1024² image masters, so the existing ceiling is ample).

## 8. New skill — `.claude/skills/audio/SKILL.md`

Flow:
1. **Derive audio profile** — read `concept.art_direction` + theme → `audio_system.mood_prompt` + `style_descriptors` (first-class, the audio analog of `visual_system.style`).
2. **Build event→SFX map** — from `concept.core_loop` + entities; record in `audio_pass.events[]`.
3. **Author recipes** — one per SFX and one music recipe, sharing the mood scaffold for coherence.
4. **Generate** — `gen-audio` each clip → committed file.
5. **Import** — set Godot import flags; `loop`/`loop_offset` for music, one-shot for SFX.
6. **Wire** — add `AudioStreamPlayer` node(s); `play()` on the mapped signals; music autoplays on scene ready.
7. **Record** `audio_pass`; set status `scored`.
8. **IP-safety** — prompt-hygiene rule (no artist names, no "in the style of <copyrighted track>"); negative-prompt guard; documented.
9. **Determinism note** — file canonical, recipe = provenance.
10. **Mixed-method honesty** — if some events lack SFX, say so explicitly in the pass.

## 9. Validator extension — `.claude/skills/validator/SKILL.md`

New method confirming:
- audio files exist and import without error;
- `AudioStreamPlayer` nodes reference valid streams;
- via `selftest.gd`, that `play()` fires on the mapped events;
- mobile sanity (file size, format, sample-rate reasonable).

## 10. Tests — `tools/comfy.test.mjs`

Mirror the raster test coverage; all run **without a GPU** via injected `fetch`:
- audio recipe `injectRecipe` `%token%` correctness;
- audio-template selection by `kind`;
- mocked HTTP returning fake audio bytes → file written;
- loud-failure-writes-no-file on each guarded endpoint.

## 11. Production specifics

- **Model:** Stable Audio Open 1.0 in ComfyUI (native audio support; text→audio, stereo 44.1kHz, clips up to ~47s — covers both short SFX and a loopable music bed).
- **SFX:** `.wav`, ~0.3–2s one-shots.
- **Music:** `.ogg` Vorbis, ~20–40s, looped via Godot import `loop`/`loop_offset`. **Seamless looping is imperfect for generative output — accepted as a known milestone limitation;** recipes target loop-friendly content (steady texture, no hard intro/outro).
- **IP-safety:** negative-prompt + prompt-hygiene rule, in the skill.

Core evidence: one audio system, two style-distinct sonic identities — the audio analog of the visual proof.
